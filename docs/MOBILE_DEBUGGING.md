# Отладка на мобильных устройствах

## Как узнать IP-адрес телефона (Samsung S23 Ultra)

### Способ 1: Через настройки телефона

1. Откройте **Настройки** → **Подключения** → **Wi‑Fi**
2. Нажмите на активную Wi‑Fi сеть (ту, к которой подключены)
3. Прокрутите вниз — там будет **IP-адрес** (например, `192.168.1.100`)

### Способ 2: Через приложение

Установите приложение **"IP Tools"** или **"Network Info"** из Play Store — они покажут IP-адрес.

### Способ 3: Через командную строку на компьютере

Если телефон подключен к той же Wi‑Fi сети, что и компьютер:

**Windows:**
```powershell
arp -a | findstr "192.168"
```

**Linux/Mac:**
```bash
arp -a | grep "192.168"
```

## Как узнать IP-адрес компьютера

### Windows:

1. Откройте **командную строку** (cmd) или **PowerShell**
2. Введите: `ipconfig`
3. Найдите **"IPv4-адрес"** в разделе вашего Wi‑Fi адаптера
   - Например: `192.168.1.50`

### Linux/Mac:

```bash
ifconfig | grep "inet "
```

или

```bash
ip addr show
```

## Как подключить телефон к dev-серверу

### Шаг 1: Убедитесь, что телефон и компьютер в одной Wi‑Fi сети

### Шаг 2: Запустите dev-сервер

```bash
npm run dev
```

Vite автоматически покажет:
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:3000/
  ➜  Network: http://192.168.1.50:3000/
```

### Шаг 3: Откройте на телефоне

Откройте в браузере телефона URL из строки **"Network"**:
```
http://192.168.1.50:3000/
```

**Важно:** Используйте IP-адрес из строки "Network", а не "localhost"!

## Как узнать порт

Порт по умолчанию: **3000**

Если порт занят, Vite автоматически выберет другой (например, 3001, 3002 и т.д.)

Проверить, какой порт используется:
- Посмотрите в консоли после запуска `npm run dev`
- Или в `vite.config.ts` указан порт: `port: 3000`

## Проблемы и решения

### Телефон не может подключиться

1. **Проверьте, что телефон и компьютер в одной Wi‑Fi сети**
2. **Проверьте файрвол Windows:**
   - Откройте "Брандмауэр Защитника Windows"
   - Разрешите доступ для Node.js или порт 3000
3. **Проверьте, что dev-сервер запущен и показывает Network URL**

### Ошибка "Connection refused"

- Убедитесь, что в `vite.config.ts` установлено `host: true`
- Проверьте, что порт не заблокирован файрволом

### Использование Eruda для отладки

После подключения к dev-серверу:
1. Eruda автоматически появится (в режиме разработки)
2. Или добавьте `?debug=true` в URL: `http://192.168.1.50:3000/?debug=true`
3. Нажмите на иконку Eruda внизу экрана для открытия консоли

## Альтернативный способ: USB-отладка

Если Wi‑Fi не работает, используйте USB-отладку:

1. Подключите телефон к компьютеру через USB
2. Включите "Отладка по USB" в настройках разработчика
3. В Chrome на компьютере откройте `chrome://inspect`
4. Найдите телефон и нажмите "Inspect"
